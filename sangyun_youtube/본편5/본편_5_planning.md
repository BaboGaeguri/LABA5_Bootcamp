## 2. 기획 핵심

| 항목 | 본편5 적용 |
|------|---------|
| **중심 갈등** | "이 18만원짜리 플라스틱 팔이 진짜 쓸만할까?" |
| **감정 곡선** | 의심("장난감 아니야?") → 조립하며 신뢰 쌓임 → 리더-팔로워 작동에 "오 되네!" → LeRobot 생태계 알고 놀람("이게 다 데이터 노림수였다고?") → EP6 기대감 |
| **정보 밀도** | SO-101 스펙 · LeRobot · 리더-팔로워 구조는 화면+자막+짧은 나레이션. HuggingFace 배경은 손그림/그래픽으로 압축. |
| **엔딩 여운** | "이 팔이 데이터를 모아줄 거예요. 다음 편은 그 데이터로 AI를 가르치는 이야기." |

## 3. 고재영 스타일 적용

| 특징 | 본편5 적용 방식 |
|------|-------------|
| **모든 건 독백** | 팀원 대화 없이 나레이션으로만 전달. 시청자에게 직접 말 거는 구조. |
| **대주제 계속 강조** | "18만원짜리 팔이 만드는 변화"를 섹션마다 한 번씩 환기. |
| **소주제 여러 개** | So-arm 소개 → LeRobot 생태계 → 부품&조립 → 첫 작동 체험 순서로 흐름. |
| **나레이션 + 영상** | 언박싱·조립·체험 장면 위에 나레이션 덧붙임. |
| **한 번 보여준 건 스킵** | 조립 과정은 핵심 단계만 자세히, 반복 작업은 타임랩스로 압축. |
| **대조 구성** | 산업용 로봇팔(수천만원) vs SO-101(18만원) 가격 대조. |
| **느낀 점** | 리더-팔로워 첫 작동 직후 팀원 반응 + "이게 데이터가 된다" 복선으로 처리. |
| **문장 사이사이 컷편집** | 나레이션 말하는 장면은 모든 문장 사이 컷. 여백 없음. |

## 4. 섹션 구조

### 핵심 섹션 9개 (+ 인트로)

| # | 핵심 섹션 | 역할 | 고민 포인트 |
|---|---------|------|-----------|
| 1 | **훅** | 가격 반전으로 "이게 뭐지?" 유발. | 산업용 로봇팔 이미지/그래픽을 어떻게 확보·표현할지. |
| - | **시리즈 공통 인트로** | 채널 로고/타이틀. | 훅의 긴장감을 끊지 않게 5초 이내로. |
| 2 | **프로젝트 리마인드** | 처음 보는 시청자를 위한 컨텍스트 + EP3(다리) 연결. | 너무 길면 이탈. 10~20초 이내로 압축. |
| 3 | **So-arm이 뭔데?** | SO-101 정체, 오픈소스, 가격, HuggingFace 반전 떡밥. | "AI 회사가 왜 로봇팔을?" 의문으로 다음 섹션 연결. |
| 4 | **LeRobot 생태계 & 체험 모델들** | HuggingFace의 큰 그림(데이터 생태계) + 커뮤니티가 만든 다른 SO-arm 활용 사례. | 커뮤니티 영상 사용 시 출처 표기. 너무 길면 강의처럼 느껴짐. |
| 5 | **부품 소개 + so-arm 조립** | 프린팅·모터·드라이버·카메라 소개 + so-arm 조립 완성. | 카메라 위치는 EP6/EP9 복선이므로 톤을 가볍게. |
| 6 | **캘리브레이션 & 텔레오퍼레이션** | 소프트웨어 연결 + 리더-팔로워 첫 작동 체험 + 데이터 수집 구조 설명. | 현장음 살리고 "이게 데이터가 된다"는 복선 심기. |
| 7 | **모터 물리 디버깅** | 실제 조립 후 마주친 모터 문제(토크·각도·케이블)와 해결 과정. | 너무 기술적으로 빠지지 않게, 감정선(좌절→해결) 위주로. |
| 8 | **ACT** | 수집한 데이터로 ACT 모델 학습 → 자율 작동 결과. | 성공/실패 둘 다 보여줘야 신뢰감 상승. 너무 길면 강의. |
| 9 | **마무리** | 소감 + "이 팔이 만들어낸 데이터가 AI를 가르친다" 복선 → EP6 예고. | 여운 남기고 끊기. 엔드스크린까지 긴장감 유지. |
