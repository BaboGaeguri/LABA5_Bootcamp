## 2. 기획 핵심

| 항목 | EP4 적용 |
|------|---------|
| **중심 갈등** | "실물 로봇 없이 시뮬레이터만으로 진짜 개발이 가능할까?" |
| **감정 곡선** | 신기함("이게 돼?") → 점점 복잡해짐 → 미로 실패 반복 → 성공 → 복선의 긴장감 |
| **정보 밀도** | ROS2·URDF 개념은 나레이션 한 줄 + 화면으로 처리. 설명 최소화. |
| **엔딩 여운** | "시뮬에서 된다고 실물에서 되는 게 아니래요. 그건 나중에 알게 돼요." |

## 3. 고재영 스타일 적용

| 특징 | EP4 적용 방식 |
|------|-------------|
| **모든 건 독백** | 팀원 대화 없이 나레이션으로만 전달. 시청자에게 직접 말 거는 구조. |
| **대주제 계속 강조** | "실물 없이도 된다"를 섹션마다 한 번씩 환기. |
| **소주제 여러 개** | ROS2 도입 → URDF → 라이다 → 미로 탈출 순서로 흐름. |
| **나레이션 + 영상** | 터미널 치는 장면, 시뮬 돌아가는 장면 위에 나레이션 덧붙임. |
| **한 번 보여준 건 스킵** | ROS2 명령어 치는 장면은 처음만 자세히. 이후 반복은 컷편집. |
| **실패 몽타주** | 미로 실패 3~5개를 빠른 컷으로 쌓아서 긴장감 유지. |
| **느낀 점** | 미로 성공 직후 팀원 표정 + 복선 멘트로 처리. |
| **문장 사이사이 컷편집** | 나레이션 말하는 장면은 모든 문장 사이 컷. 여백 없음. |

## 4. 섹션 구조

### 핵심 세션 7개

| # | 핵심 세션 | 역할 | 고민 포인트 |
|---|---------|------|-----------|
| 1 | **훅** | 결말 먼저 보여주기. "이게 뭐지?" 유발. | 하이리온 → Gazebo 성공 → "실물 없이요" 3단 구조로 빠르게. |
| 2 | **윈도우 밀어버린 사건** | 감정 입구. 시청자 웃기고 공감 끌어내기. | 사건 자체가 강력 — 짧고 빠르게 터뜨릴 것. |
| 3 | **프로젝트 소개** | 처음 보는 시청자를 위한 컨텍스트. "이게 뭔지" 20초 안에 정리. | 너무 길면 이탈. 2클립 이내로 압축. |
| 4 | **ROS2 + URDF 개념 설명** | 정보 전달 구간. | **화면녹화(실제 작동) + 손그림(개념 설명)** 조합. 개념당 30초 이내 목표. |
| 5 | **잘 안 되는 포인트들** | 긴장감 + 도파민 장치. 실패를 쌓아서 성공을 크게 만듦. | 실패 종류 다양하게. 같은 실패 반복은 컷편집 스킵. |
| 6 | **성공 시연** | 감정 해소. 영상의 절정. | 성공 순간 현장음 살리기. 팀원 반응 필수. |
| 7 | **AI로 하니까 소프트웨어는 훅훅 지나간다!!** | 깨달음 + 엔딩 여운. 다음 편 연결. | 이 멘트가 대주제를 회수하는 순간. 담담하게 말할수록 임팩트. |

## 6. 타임라인 트랙 구성

| 트랙 | 용도 | EP4 적용 |
|------|------|---------|
| **Video 1** | 메인 영상 | 현장 촬영(팀원 작업) + 화면녹화(터미널·Gazebo) |
| **Video 2** | 오버레이 | 손그림 설명, 챕터 타이틀("Phase 3: 시뮬레이션"), 자막 그래픽 |
| **Audio 1 — BGM** | 배경음악 | 테크 인스트루멘탈. 볼륨 낮게. 장면 전환 시 페이드인/아웃. |
| **Audio 2 — SFX** | 효과음 | 훅 임팩트 컷, 성공 비트 드롭, 실패 컷 효과음에만 사용. |
| **Audio 3 — VO** | 나레이션 | 독백. 볼륨 가장 높게. VO 구간에서 BGM 덕킹 적용. |

> **덕킹 포인트:** VO 나오는 모든 구간에서 BGM을 -10~-15dB 자동 감소.
