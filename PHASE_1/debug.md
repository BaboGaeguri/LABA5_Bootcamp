## [2026-03-19] 서보 모터 테스트

### 환경
- 실행 파일: `1-5_motor_test.py`
- 환경: 희승 컴퓨터 → 학교 유선 랜 → 라즈베리파이 SSH

### 증상
- `STEP = 5` (5도 단위)로 설정했으나 실제로 약 30도 누적 입력이 있어야 서보가 움직임
- 반응 속도가 느림

### 원인 추정
- 펄스폭 범위(`min_pulse_width`, `max_pulse_width`)가 실제 서보 특성과 불일치
- 기존 설정 `0.0005~0.0025`(0.5ms~2.5ms)가 너무 넓어서 작은 각도 변화가 물리적으로 무의미

### 시도한 수정

| 항목 | 기존 값 | 변경 값 |
|------|---------|---------|
| `min_pulse_width` | 0.0005 | 0.001 |
| `max_pulse_width` | 0.0025 | 0.002 |
| `STEP` | 5 | 10 |

### 결과
- 펄스폭 `0.001~0.002` + `STEP=10` 으로 변경 후 서보 정상 동작 확인

## [2026-03-20] 서보 외부 전원 시도 (배터리)

### 시도
- 라파 5V 전원 부족 가능성으로 AA 배터리 외부 전원 시도
- 3구 홀더 2개 직렬 연결, 배터리 4개(3+1) 장착

### 문제
- 3구 홀더에 빈 칸 2개 존재 → 회로 단락으로 전압 0V
- 빈 칸을 채우면 6개 직렬 = 9V → SG90 최대 허용 전압(6V) 초과
- 결론: 배터리 구성으로는 정상 전압(6V) 구성 불가 → **배터리 교체 포기**
- 라파 5V핀으로 복귀

### 서보 펄스폭 재조정
- 증상: 10도 단위 명령에 20도 이상 누적해야 서보 반응
- 원인: `0.001~0.002` 범위가 실제 서보 유효 범위보다 좁음
- 해결: `0.0005~0.0025`로 복원

| 항목 | 변경 전 | 변경 후 |
|------|---------|---------|
| `min_pulse_width` | 0.001 | 0.0005 |
| `max_pulse_width` | 0.002 | 0.0025 |

### 결과
- 변경 전보다는 훨씬 나아졌으나 여전히 움직여야 할 때 움찔거리다 멈추는 현상 발생
- 전류 부족(MG90S 스톨 전류 ~700mA, 라파 5V 핀 최대 ~300mA) 가능성으로 배터리 외부 전원 재시도

## [2026-03-20] 전원 역류 사고

### 발생 경위
- 라파 5V 핀에 서보 빨간선이 연결된 상태에서 AA 배터리(6V) 동시 연결
- `AA 배터리 6V → 서보 빨간선 → 브레드보드 + 레일 → 라파 5V 핀` 으로 역류
- 라파 입장에서 외부 6V가 밀고 들어오는 상황 → 전원 경고등 점등

### 교훈 - 외부 전원 연결 순서
1. **라파 5V 핀에서 서보 빨간선 먼저 분리**
2. 그 다음 외부 전원(배터리) 연결
- 항상 라파 연결을 먼저 끊고 외부 전원을 연결할 것

### 현재 상태
- 라파 정상 동작 확인, SSH 연결 OK
- 배터리 외부 전원 시도 2회 모두 실패 → **배터리 방향 포기**
- 라파 5V 핀으로 계속 진행

## [2026-03-20] pigpio 적용 시도 및 철회

### 시도
- `gpiozero AngularServo` 소프트웨어 PWM 불안정 문제로 pigpio 하드웨어 PWM 교체 시도
- `python3-pigpio` 설치, 소스 빌드 완료

### 문제
- `sudo pigpiod -g` 실행 시 아래 에러:
  ```
  unknown rev code (d04171)
  Sorry, this system does not appear to be a raspberry pi.
  ```
- 원인: **pigpio는 라즈베리파이 5를 지원하지 않음**
- 리비전 코드 `d04171` = Raspberry Pi 5

### 결론
- Pi 5에서 GPIO 라이브러리 지원 현황:
  - `pigpio` → Pi 4까지만 지원, **Pi 5 미지원**
  - `lgpio` → Pi 5 전용 후속 라이브러리
  - `gpiozero` → Pi 5에서 내부적으로 lgpio를 백엔드로 자동 사용
- **gpiozero로 복귀** (이미 작동 확인된 방식)

## [2026-03-20] 파란색 카드 추적 - 서보 방향 반전

### 증상
- 파란색 물체를 따라가야 하는데 오히려 피하는 방향으로 서보가 움직임

### 원인
- 서보 장착 방향에 따라 `error * Kp` 부호가 반대로 작용
- error > 0 (물체가 오른쪽) → 서보가 왼쪽으로 회전해야 하는데 오른쪽으로 회전

### 해결
```python
# 변경 전
movement = clamp(error * Kp, -MAX_STEP, MAX_STEP)
# 변경 후
movement = clamp(-error * Kp, -MAX_STEP, MAX_STEP)
```

## [2026-03-20] 파란색 카드 추적 - 서보 진동 문제

### 증상
- 파란색 물체 인식 중 서보가 좌우로 계속 진동
- 물체가 없어도 배경 파란색을 인식해 서보가 움직임

### 원인 분석
- **P제어기 진동(Oscillation)**: 물체가 오른쪽 → 서보 왼쪽 이동 → 물체가 왼쪽으로 보임 → 서보 오른쪽 이동 → 반복
- 매 프레임(~10fps)마다 서보가 반응해 오버슈팅 발생
- 배경 파란색 오인식으로 물체 없을 때도 진동

### 시도한 수정

| 항목 | 변경 내용 | 효과 |
|------|----------|------|
| `Kp` | 0.06 → 0.02 | 부분 개선 |
| `MAX_STEP` | 4 → 2 | 부분 개선 |
| `MIN_AREA` | 8000 → 15000 | 배경 노이즈 일부 감소 |
| `BLUE_LOWER` S채도 | 120 → 140 → 180 | 배경 오인식 감소 |
| error 이동평균 (5프레임) | `smooth_error` 적용 | 진동 완화 시도 중 |

### 현재 상태
- 이동평균 적용 후 테스트 진행 중
- 근본 해결 안 되면 ③ 서보 업데이트 주기 제한 또는 MediaPipe 전환 검토
