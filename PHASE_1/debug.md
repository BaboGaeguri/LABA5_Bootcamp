## [2026-03-19] 서보 모터 테스트

### 환경
- 실행 파일: `1-5_motor_test.py`
- 환경: 희승 컴퓨터 → 학교 유선 랜 → 라즈베리파이 SSH

### 증상
- `STEP = 5` (5도 단위)로 설정했으나 실제로 약 30도 누적 입력이 있어야 서보가 움직임
- 반응 속도가 느림

### 원인 추정
- 펄스폭 범위(`min_pulse_width`, `max_pulse_width`)가 실제 서보 특성과 불일치
- 기존 설정 `0.0005~0.0025`(0.5ms~2.5ms)가 너무 넓어서 작은 각도 변화가 물리적으로 무의미

### 시도한 수정

| 항목 | 기존 값 | 변경 값 |
|------|---------|---------|
| `min_pulse_width` | 0.0005 | 0.001 |
| `max_pulse_width` | 0.0025 | 0.002 |
| `STEP` | 5 | 10 |

### 결과
- 펄스폭 `0.001~0.002` + `STEP=10` 으로 변경 후 서보 정상 동작 확인

## [2026-03-20] 하드웨어 전체 테스트 (`hardware_test.py`)

### 테스트 항목 및 결과

| 장치 | 결과 | 비고 |
|------|------|------|
| 초록 LED | 정상 | `g` 키 토글 |
| 빨간 LED | 정상 | `r` 키 토글 |
| 부저 | 정상 | 수동 부저 → `RPi.GPIO PWM 1000Hz`로 해결 |
| 서보 모터 | 정상 | `←/→` 키, `c` 중앙 복귀 |

### 부저 이슈 및 해결

- **증상**: `gpiozero Buzzer` 클래스로 소리 안 남
- **원인**: 아두이노 키트 부저는 수동 부저(passive buzzer) — PWM 신호 필요
- **해결**: `RPi.GPIO.PWM(22, 1000)` 으로 교체, duty cycle 50% 적용

## [2026-03-20] 서보 외부 전원 시도 (배터리)

### 시도
- 라파 5V 전원 부족 가능성으로 AA 배터리 외부 전원 시도
- 3구 홀더 2개 직렬 연결, 배터리 4개(3+1) 장착

### 문제
- 3구 홀더에 빈 칸 2개 존재 → 회로 단락으로 전압 0V
- 빈 칸을 채우면 6개 직렬 = 9V → SG90 최대 허용 전압(6V) 초과
- 결론: 배터리 구성으로는 정상 전압(6V) 구성 불가 → **배터리 교체 포기**
- 라파 5V핀으로 복귀

### 서보 펄스폭 재조정
- 증상: 10도 단위 명령에 20도 이상 누적해야 서보 반응
- 원인: `0.001~0.002` 범위가 실제 서보 유효 범위보다 좁음
- 해결: `0.0005~0.0025`로 복원

| 항목 | 변경 전 | 변경 후 |
|------|---------|---------|
| `min_pulse_width` | 0.001 | 0.0005 |
| `max_pulse_width` | 0.002 | 0.0025 |

### 결과
- 확인 중

## [2026-03-20] 배선 재정리

### 현재 배선 상태

| 구성요소 | 연결 경로 |
|---------|----------|
| 초록 LED | GPIO17 (11번) → 저항 → 초록LED(+) → 초록LED(-) → GND (6번) |
| 빨간 LED | GPIO27 (13번) → 저항 → 빨간LED(+) → 빨간LED(-) → GND (6번) |
| 부저 | GPIO22 (15번) → 부저(+) → 부저(-) → GND (6번) |
| 서보 전원 | 5V (2번) → +레일 → 서보(빨강) → 서보(갈색) → GND (6번) |
| 서보 신호 | GPIO18 (12번) → 서보(주황) |
